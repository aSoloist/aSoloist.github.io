Python requests库的使用

发表于 2018-03-14 | 分类于 Python

安装

如果你的系统安装的有pip，只需要在终端中运行pip install requests即可。

也可以通过git clone git://github.com/kennethreitz/requests.git获取源码。

使用

发送请求

第一步需要导入模块：

import requests

之后可以使用get()函数获取某个网页。

response = requests.get('http://www.soloist.top')

这样，我们就有一个response对象，通过这个对象我们可以获取所有我们想要的信息。

同理，我们可以用post()函数发送一个POST请求:

response = requests.post('http://www.example.com', data = {'key':'value'})

PUT、DELETE、HEAD、OPTIONS这些类型的请求与上面相似。

响应内容

通过text属性，可以读取服务器响应的内容：

requests可以自动解码来自服务器的内容，大多数Unicode字符集都可以无缝解码。

发出请求后，requests会根据HTTP请求头推测响应内容的编码，我们可以使用encoding属性获取requests使用的编码，同时也可以修改编码：

response.encoding = 'gbk'

如果你改变了编码，每当你访问text.requests都将会使用encoding的新值。你可能希望在使用特殊逻辑计算出文本的编码的情况下来修改编码。比如HTTP和XML自身可以指定编码。这样的话，你应该使用content来找到编码，然后设置encoding为相应的编码。这样就能使用正确的编码解析text了。

如果JSON解码失败，json()就会抛出一个异常。例如，响应内容是401 (Unauthorized)，尝试调用json()将会抛出ValueError: No JSON object could be decoded异常。

需要注意的是，成功调用json() 并不意味着响应的成功。有的服务器会在失败的响应中包含一个JSON对象（比如HTTP 500的错误细节）。这种JSON会被解码返回。要检查请求是否成功，请使用raise_for_status()或者检查status_code是否和你的期望相同。
